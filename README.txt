Carpeta SolucionArbol:
solucion1.py: Contiene la implementación principal del algoritmo que trabaja con estructuras tipo árbol.
 Aquí se define la lógica para resolver el problema planteado usando árboles.

Test.txt: Archivo de prueba, contiene entradas de datos para validar el funcionamiento del algoritmo
en solucion1.py.

Carpeta SolucionArray:
app.py: Archivo principal que orquesta la ejecución del programa. Contiene el punto de entrada
y la lógica general para coordinar los módulos.

clases.py: Define las clases necesarias para la solución basada en arreglos. Contiene la estructura
de datos y métodos asociados.

estadisticas.py: Módulo encargado de calcular estadísticas relacionadas con el procesamiento de los
arreglos (tiempos, comparaciones, etc.).

lectura.py: Maneja la entrada de datos. Este módulo prepara los datos para su posterior análisis.

merge.py: Implementa algoritmos de combinación de datos o estructuras, relacionados con ordenamiento 
o fusión de resultados.

Test.txt: Archivo de entrada de pruebas, usado para verificar el comportamiento de los algoritmos
desarrollados en esta carpeta. 

informe.pdf: Documento que contiene el análisis del problema, la explicación de las soluciones implementadas
y los resultados obtenidos.

README.txt: Archivo de texto con instrucciones de uso, estructura del proyecto y cualquier otra información
relevante para entender y ejecutar el proyecto.

INSTRUCCIONES:

Para la ejecución de la solucion1.py. Para esto, debe ubicarse dentro de lar carpeta SolucionArbol, ahí ejecutar
el programa. Le solicitará ingresar el nombre del archivo que contiene los datos de entrada, posteriormente general
un archivo de salida con los resultados. De igual forma para la solución 2 solo que esta vez debe estar ubicado dentro
la carpeta SolucionArray. Al momento de ingresar el nombre del archivo debe ingresar la ruta o en su defecto, si se 
encuentra dentro de la carpeta, únicamente debe ingresar el nombre.
